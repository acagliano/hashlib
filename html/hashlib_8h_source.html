42/*
43Cryptographically-Secure Random Number Generator (CS-RNG)
44
45Many of the psuedorandom number generators (PRNGs) you find in computers and
46even the one within the C toolchain for the CE are insecure for cryptographic
47purposes. They produce statistical randomness, but the state is generally seeded
48using a value such as rtc_Time(). If an adversary reconstructs the seed, every
49output of the PRNG becomes computable with little effort. These types of PRNGs
50are called deterministic algorithms--given the input, the output is predictable.
51These PRNGs work for games and other applications where the illusion of randomness
52is sufficient, but they are not safe for cryptography.
53
54A secure PRNG is an random number generator that is not only statistically random, but
55also passes the next-bit and state compromise tests. The _next-bit test_ is defined like so:
56given the prior output of the PRNG (bits 0=>i), the next bit (i+1) of the output cannot be
57predicted by a polynomial-time statistical test with a probability non-negligibly greater than 50%.
58In simpler terms, a secure PRNG must be unpredictable, or "entropic".
59The _state compromise test_ means that an adversary gaining knowledge of the initial state of
60the PRNG does not gain any information about its output.
61
62The SRNG previded by HASHLIB solves both tests like so:
63(next-bit)
64 <> The SRNG's output is derived from a 119-byte entropy pool created by reading
65 data from the most entropic byte located within floating memory on the device.
66 <> The "entropic byte" is a byte containing a bit that that, out of 1024 test reads,
67 has the closest to a 50/50 split between 1's and 0's.
68 <> The byte containing that bit is read in xor mode seven times per byte to offset
69 any hardware-based correlation between subsequent reads from the entropic byte.
70 <> The entropy pool is then run through a cryptographic hash to spread that entropy
71 evenly through the returned random value.
72 <> The SRNG produces 96.51 bits of entropy per 32 bit number returned.
73 <> Assertion: A source of randomness with sufficient entropy passed through a cryptographic hash
74 will produce output that passes all statistical randomness tests as well as the next-bit test.
75(state compromise)
76 <> The entropy pool is discarded after it is used once, and a new pool is
77 generated for the next random number.
78 <> ^ This means that the prior state has no bearing on the next output of the PRNG.
79 <> The SRNG destroys its own state after the random number is generated so that
80 the state used to generate it does not persist in memory.
81
82* Due to the derivation of entropy from subtle variations in the electrical state of
83 unmapped memory, this SRNG can also be considered a _hardware-based RNG_ (HWRNG).
84*/

120/*
121Cryptographic Hashes
122
123A cryptographic hash is used to validate that data is unchanged between two endpoints.
124It is similar to a checksum, but checksums can be easily fooled; cryptographic hashes
125are a lot harder to fool due to how they distribute the bits in a data stream.
126The general use of a hash is as follows: the party sending a message hashes it and
127includes that hash as part of the message. The recipient hashes the message (except the hash)
128themselves and then compares that hash to the one included in the message. If the hashes match,
129the message is complete and unaltered. If the hashes do not match, the message is incomplete
130or has been tampered with.
131*/

322/*
323Advanced Encryption Standard (AES)
324
325AES is a form of symmetric encryption, and also is a form of block cipher.
326Symmetric encryption means that the same key works in both directions.
327A block cipher is an encryption algorithm that operates on data in segments (for AES, 16 bytes),
328moving through it one segment at a time.
329
330Symmetric encryption is usually fast, and is generally more secure for smaller key sizes.
331AES is one of the most secure encryption systems in use today.
332The most secure version of the algorithm is AES-256 (uses a 256-bit key).
333AES is one of the open-source encryption schemes believed secure enough to withstand even
334the advent of quantum computing.
335*/

481/*
482 RSA Public Key Encryption
483
484 Public key encryption is a form of asymmetric encryption.
485 This means that a key only works in one direction, and both parties need a public key
486 and a private key. The private key is used to decrypt a message and the public key is
487 used to encrypt.
488 In RSA, the public and private keys are modular inverses of each other, such that:
489 encrypted = message ** public exponent % public key, and
490 message = encrypted ** private exponent % private modulus
491 ** means power, % means modulus
492
493 65537 (and a few other Fermat primes) are commonly used as public exponents.
494 The public key (modulus) is sent in the clear and is known to to everyone.
495 The cryptographic strength of RSA comes from the difficulty of factoring the modulus.
496 Asymmetric encryption is generally VERY slow. Using even RSA-1024 on the TI-84+ CE will
497 take several seconds. For this reason, you usually do not use RSA for sustained encrypted
498 communication. Use RSA to share a symmetric key, and then use AES for future messages.
499
500 */
