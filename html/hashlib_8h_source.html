43/*
44Secure Psuedorandom Number Generator (SPRNG)
45
46Many of the psuedorandom number generators (PRNGs) you find in computers and
47even the one within the C toolchain for the CE are insecure for cryptographic
48purposes. They produce statistical randomness, but the state is generally seeded
49using a value such as rtc_Time(). If an adversary reconstructs the seed, every
50output of the PRNG becomes computable with little effort. These types of PRNGs
51are called deterministic algorithms--given the input, the output is predictable.
52These PRNGs work for games and other applications where the illusion of randomness
53is sufficient, but they are not safe for cryptography.
54
55A secure PRNG is an random number generator that is not only statistically random, but
56also passes the next-bit and state compromise tests. The _next-bit test_ is defined like so:
57given the prior output of the PRNG (bits 0=>i), the next bit (i+1) of the output cannot be
58predicted by a polynomial-time statistical test with a probability non-negligibly greater than 50%.
59In simpler terms, a secure PRNG must be unpredictable, or "entropic".
60The _state compromise test_ means that an adversary gaining knowledge of the initial state of
61the PRNG does not gain any information about its output.
62
63The PRNG previded by HASHLIB solves both tests like so:
64(next-bit)
65 <> The PRNG's output is derived from a 119-byte entropy pool created by reading
66 data from the most entropic byte located within floating memory on the device.
67 <> The "entropic byte" is a byte containing a bit that that, out of 1024 test reads,
68 has the closest to a 50/50 split between 1's and 0's.
69 <> The byte containing that bit is read in xor mode seven times per byte to offset
70 any hardware-based correlation between subsequent reads from the entropic byte.
71 <> The entropy pool is then run through a cryptographic hash to spread that entropy
72 evenly through the returned random value.
73 <> The PRNG produces 96.51 bits of entropy per 32 bit number returned.
74 <> Assertion: A source of randomness with sufficient entropy passed through a cryptographic hash
75 will produce output that passes all statistical randomness tests as well as the next-bit test.
76(state compromise)
77 <> The entropy pool is discarded after it is used once, and a new pool is
78 generated for the next random number.
79 <> ^ This means that the prior state has no bearing on the next output of the PRNG.
80 <> The PRNG destroys its own state after the random number is generated so that
81 the state used to generate it does not persist in memory.
82*/

115/*
116SHA-256 Cryptographic Hash
117
118A cryptographic hash is used to validate that data is unchanged between two endpoints.
119It is similar to a checksum, but checksums can be easily fooled; cryptographic hashes
120are a lot harder to fool due to how they distribute the bits in a data stream.
121The general use of a hash is as follows: the party sending a message hashes it and
122includes that hash as part of the message. The recipient hashes the message (except the hash)
123themselves and then compares that hash to the one included in the message. If the hashes match,
124the message is complete and unaltered. If the hashes do not match, the message is incomplete
125or has been tampered with.
126*/

270/*
271Advanced Encryption Standard (AES)
272
273AES is a form of symmetric encryption, and also is a form of block cipher.
274Symmetric encryption means that the same key works in both directions.
275A block cipher is an encryption algorithm that operates on data in segments (for AES, 16 bytes),
276moving through it one segment at a time.
277
278Symmetric encryption is usually fast, and is generally more secure for smaller key sizes.
279AES is one of the most secure encryption systems in use today.
280The most secure version of the algorithm is AES-256 (uses a 256-bit key).
281AES is one of the open-source encryption schemes believed secure enough to withstand even
282the advent of quantum computing.

447/*
448 RSA Public Key Encryption
449
450 Public key encryption is a form of asymmetric encryption.
451 This means that a key only works in one direction, and both parties need a public key
452 and a private key. The private key is used to decrypt a message and the public key is
453 used to encrypt.
454 In RSA, the public and private keys are modular inverses of each other, such that:
455 encrypted = message ** public exponent % public key, and
456 message = encrypted ** private exponent % private modulus
457 ** means power, % means modulus
458
459 65537 (and a few other Fermat primes) are commonly used as public exponents.
460 The public key (modulus) is sent in the clear and is known to to everyone.
461 The cryptographic strength of RSA comes from the difficulty of factoring the modulus.
462 Asymmetric encryption is generally VERY slow. Using even RSA-1024 on the TI-84+ CE will
463 take several seconds. For this reason, you usually do not use RSA for sustained encrypted
464 communication. Use RSA to share a symmetric key, and then use AES for future messages.
465
466 */
